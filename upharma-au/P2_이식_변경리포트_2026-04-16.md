### 3.3 `static/app.js`

**변경 위치:** 파일 끝

**구조:**

```
1~938줄     : 호주 원본 JS 그대로 유지
954줄~      : [호환 shim(매개층) 시작]
              - _escHtml      → _escapeHtml 로 위임
              - _loadReports  → reportStore 로 위임  
              - _setText      → 신규 정의
978~1644줄 : [팀원 원본 p2 JS 블록 — 수정 없이 그대로]
1645줄     : [팀원 원본 p2 JS 블록 끝]
1647~1653줄: DOMContentLoaded 업데이트
              → initP2Strategy() 호출 추가
```

**호환 shim(매개층)이 필요한 이유:**

팀원 코드와 호주 기존 코드에서 같은 역할을 하는 함수 **이름이 다름**. 팀원 코드를 한 글자도 안 건드리기로 했으니, 팀원이 쓰는 이름을 호주 코드의 이름으로 "연결만" 해주는 얇은 래퍼(wrapper)를 넣음.

| 팀원이 쓰는 이름 | 호주 원본 이름 | shim 처리 |
|---|---|---|
| `_escHtml` | `_escapeHtml` | 팀원 이름을 호주 이름으로 연결 |
| `_loadReports` | `reportStore` | 팀원 이름을 호주 보고서 저장소로 연결 |
| `_setText` | (없음) | 신규 정의 (DOM에 텍스트 쓰는 보조 함수) |

`typeof` 가드로 감싸놔서, 호주 원본 함수가 있을 때만 shim이 붙음. 호주 원본을 덮어쓰지 않음.

**이식된 팀원 JS 함수 25개 (v2.0 리네임 1개 포함):**

`_makeP2Defaults`, `initP2Strategy`, `switchP2Tab`, `setP2AiSeg`, `handleP2FileSelect`, `_showP2AiStatus`, `_resetP2AiResultView`, `runP2AiPipeline`, `_pollP2AiPipeline`, `_renderP2AiResult`, `_p2FillExchangeRate`, `_p2FillBaseFromReport`, `_syncP2ReportsOptions`, `_getP2SelectedReport`, **`_extractAudHint`** (← `_extractSgdHint` 리네임), `_renderP2ReportBrief`, `_calcP2Manual`, `_renderP2Manual`, `_p2OptionCardHtml`, `_bindP2OptionEvents`, `_renderP2CustomAddSection`, `_p2ManualScenarioReason`, `_p2ScenarioHtml`, `_renderP2PdfSection`, `_generateP2Pdf`

  if (rate > 0) {
    const divisor = 1 + rate / 100;
