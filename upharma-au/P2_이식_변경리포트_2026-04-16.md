### 3.3 `static/app.js`

**변경 위치:** 파일 끝

**구조:**

```
1~938줄     : 호주 원본 JS 그대로 유지
954줄~      : [호환 shim(매개층) 시작]
              - _escHtml      → _escapeHtml 로 위임
              - _loadReports  → reportStore 로 위임  
              - _setText      → 신규 정의
978~1644줄 : [팀원 원본 p2 JS 블록 — 수정 없이 그대로]
1645줄     : [팀원 원본 p2 JS 블록 끝]
1647~1653줄: DOMContentLoaded 업데이트
              → initP2Strategy() 호출 추가
```

**호환 shim(매개층)이 필요한 이유:**

팀원 코드와 호주 기존 코드에서 같은 역할을 하는 함수 **이름이 다름**. 팀원 코드를 한 글자도 안 건드리기로 했으니, 팀원이 쓰는 이름을 호주 코드의 이름으로 "연결만" 해주는 얇은 래퍼(wrapper)를 넣음.

| 팀원이 쓰는 이름 | 호주 원본 이름 | shim 처리 |
|---|---|---|
| `_escHtml` | `_escapeHtml` | 팀원 이름을 호주 이름으로 연결 |
| `_loadReports` | `reportStore` | 팀원 이름을 호주 보고서 저장소로 연결 |
| `_setText` | (없음) | 신규 정의 (DOM에 텍스트 쓰는 보조 함수) |

`typeof` 가드로 감싸놔서, 호주 원본 함수가 있을 때만 shim이 붙음. 호주 원본을 덮어쓰지 않음.

**이식된 팀원 JS 함수 25개:**

`_makeP2Defaults`, `initP2Strategy`, `switchP2Tab`, `setP2AiSeg`, `handleP2FileSelect`, `_showP2AiStatus`, `_resetP2AiResultView`, `runP2AiPipeline`, `_pollP2AiPipeline`, `_renderP2AiResult`, `_p2FillExchangeRate`, `_p2FillBaseFromReport`, `_syncP2ReportsOptions`, `_getP2SelectedReport`, `_extractSgdHint`, `_renderP2ReportBrief`, `_calcP2Manual`, `_renderP2Manual`, `_p2OptionCardHtml`, `_bindP2OptionEvents`, `_renderP2CustomAddSection`, `_p2ManualScenarioReason`, `_p2ScenarioHtml`, `_renderP2PdfSection`, `_generateP2Pdf`

## 4. 백엔드 API (Jisoo가 구현 예정)

팀원 JS가 호출하는 API 엔드포인트(서버 주소) 목록. 호주 프로젝트에는 아직 없음.

| 엔드포인트 | 역할 |
|---|---|
| `POST /api/p2/upload` | 1공정 보고서 PDF 업로드 |
| `POST /api/p2/pipeline` | AI 가격 분석 파이프라인 시작 |
| `GET /api/p2/pipeline/status` | 파이프라인 진행 상태 조회 (폴링) |
| `GET /api/p2/pipeline/result` | 최종 결과 조회 |
| `GET /api/p2/report` | 저장된 보고서 목록/상세 |
| `GET /api/exchange` | 실시간 환율 |
| `GET /api/report/download` | PDF 다운로드 |

**연결 방식:** Jisoo가 이전에 받은 stage2(2공정) 백엔드 작업본과 합쳐서 `render_api.py`(또는 해당 라우터)에 엔드포인트 구현. 팀원 JS가 보내는 요청/응답 포맷을 확인하면서 호주 8개 품목(Hydrine, Sereterol, Omethyl, Rosumeg, Atmeg, Ciloduo, Gastiin CR, Gadvoa) 로직과 연결.

## 6. 추후 Jisoo가 해야 할 일 (참고)

1. **백엔드 API 7개 구현** (`render_api.py` 등)
   - Jisoo가 가진 2공정 백엔드 작업본과 합치면서
   - 호주 PBS/TGA 가격 로직 연결

2. **싱가포르 → 호주 국가 문자열 치환** (눈으로 보이는 부분)
   - `SGD` → `AUD`
   - `ALPS 조달청` → 호주 공공조달 채널 명칭 (예: Health-Purchasing-Vic 등)
   - "27개 공공기관 통합구매" 설명 문구 — 호주 기준으로 재작성

3. **Jisoo가 이전에 하던 1공정 보고서** 포맷과 `/api/p2/report` 응답 스키마 맞추기
   - 팀원 JS `_p2FillBaseFromReport`가 보고서에서 가격 정보를 뽑아 씀
   - 호주 AEMP/DPMQ 값을 어떤 key로 제공할지 확인 필요

4. **PBAC 임상우월성 이슈**(Rosumeg/Atmeg), **Commercial Withdrawal**(Ciloduo), **병원조달루트**(Gadvoa/Omethyl), **함량·제형 불일치 confidence 0.6**(Omethyl) 등 호주 특유 플래그를 2공정 결과 UI에 어떻게 노출할지 — 팀원 UI에는 없는 부분이므로 추가 설계 필요

## 7. 롤백 방법 (필요 시)

`app.js`만 백업 존재:

HTML/CSS는 Edit 도구로 부분 수정이라 `.bak`이 없음. 롤백 필요하면 Git 커밋 되돌리기 권장.
